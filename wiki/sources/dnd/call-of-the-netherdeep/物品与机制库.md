# 物品与机制库

## 摘要
这份来源把《Call of the Netherdeep》里最容易散落在章节、附录、run 真相与人物页中的关键物件和机制收拢到了同一个入口里。它不只是在列清单，而是在回答这些物件到底是什么、如何获得、怎么运作、为什么重要，以及它们在 official module 与本团 campaign canon 里分别承担什么层级的解释任务。

## 这份来源实际补强了什么
### 1. 把“关键物件”从章节道具提升为可查询节点
它让下列问题不必再分别翻附录、地点页和 run 时间线来拼：
- [[wiki/entities/items/dnd/call-of-the-netherdeep/三祷之坠]] 为什么既是遗物、钥匙，又是阿利克辛主线的媒介
- [[wiki/concepts/lore/dnd/call-of-the-netherdeep/月蚀矿]] 与 [[wiki/concepts/mechanics/dnd/call-of-the-netherdeep/月蚀腐化]] 如何构成“强力装备 + 身心侵蚀”的完整风险链
- [[wiki/concepts/lore/dnd/call-of-the-netherdeep/拉克桑信标]]、[[wiki/entities/locations/dnd/call-of-the-netherdeep/提仕坦方尖碑]]、[[wiki/concepts/lore/dnd/call-of-the-netherdeep/秘迹魔法]] 与 [[wiki/concepts/lore/dnd/call-of-the-netherdeep/言语之力]] 为什么会在终局被压到同一条解释链上
- [[wiki/entities/items/dnd/call-of-the-netherdeep/潮汐透镜]]、[[wiki/entities/items/dnd/call-of-the-netherdeep/阈限之石]]、[[wiki/entities/items/dnd/call-of-the-netherdeep/沙漏之书]] 这些跨地点关键物件分别承担观测、迁移与解谜功能

### 3. 明确区分 official 物品规则与 campaign 真相装置
这份来源同时混合了两类信息：
- official module 中本就存在、可直接沉淀到 canonical 层的装备与机制
- 本团私有扩写中才成立的终局物件、时空技术、血脉媒介与真相装置

因此本次 ingest 的处理规则是：
- official 侧已有稳定页面的内容，优先深化既有 canonical 页，而不是重造重复节点
- 明显属于本团 campaign-only 的装置、媒介与真相机制，建立独立页并通过 warning callout 回链到 `run/` 真相、时间线与人物层
- 对“[[wiki/concepts/mechanics/dnd/call-of-the-netherdeep/魔法屏障与基石]] / [[wiki/entities/items/dnd/call-of-the-netherdeep/呼吸罩]]”这类已有稳定解释的官方条目，不额外制造模糊新页

## 对模糊条目的处理
### 供氧晶石
这份来源把“供氧晶石”作为一个使用者口径上的统称，而不是稳定的单一 canonical 物件。当前更可靠的拆解方式是：
- 大范围无水空间维持：[[wiki/concepts/mechanics/dnd/call-of-the-netherdeep/魔法屏障与基石]]
- 单体水下供氧装备：[[wiki/entities/items/dnd/call-of-the-netherdeep/呼吸罩]]

因此本次 ingest 不单独创建“供氧晶石”页，避免把两个不同层级的装置错误折叠成一个词条。

## 这份来源能直接支持的 query
- 哪些物件属于 official module，哪些属于本团真相层？
- 拉克桑信标、秘迹魔法、方尖碑与言语之力之间是什么关系？
- 为什么螺旋之核、蝎子王冠、玉佩和战锤都不是普通魔法物品，而是人物命运的结构件？
- 月蚀矿、月蚀腐化和月蚀装备的风险链到底如何闭环？
- 为什么“供氧晶石”不该直接做成一个独立 canonical 页？
